# Wandao v1.3.5

v1.3.5 是一轮以“恢复可用性、统一任务体验、保证导出完整性”为目标的生产修复版本，集中解决目录层级、登录会话、任务中断续跑和结果反馈中影响实际使用的问题。

## 重点修复

### 飞书 Wiki 与 Markdown 文件

- 飞书目录统一识别原生文档和 Markdown 文件。真实知识库验证由 166 篇可选恢复为 221/221 篇可选，其中包含 55 个 `obj_type=12` Markdown 文件。
- Markdown 文件优先通过飞书原文件接口下载，保留原始 UTF-8 内容、标题和 Markdown 语法；修复 `.md.md` 双扩展名。
- 健康浏览器登录态不再被旧 Cookie 覆盖；登录过期时只恢复凭证并重试一次，避免重复导航和死循环。
- 修复飞书页面长期停留在 `interactive` 时被误判超时、旧登录页面竞态、非目标标签页权限错误阻止目标导航等问题。
- 飞书未提供的相对图片资源会明确记录为资源失败并将结果标记为“部分完成”，不再静默报告完整成功；页面预览兜底同样会标记内容可能不完整。

### 知识星球中断续跑（Issue #43）

- 新一轮任务从输出目录现有最大编号继续分配，不再从 `01` 重新开始。
- 跨任务 ID 恢复历史分页游标和未完成条目，资源重试复用原 Markdown 路径，避免重复文件和编号漂移。
- 每次续跑先检查最新帖子，命中上次“时间戳 + 帖子 ID”水位线后再继续历史游标；同时间戳的不同新帖不会被跳过。
- 中断后仍生成最新报告和按运行 ID 保存的历史报告，列出本轮已导出、已跳过、待处理和失败条目。

### Obsidian Vault 归档导出（实验性）

- 新增社区实验性 Obsidian 插件，可扫描本地 Vault 的文件夹与 Markdown 层级，并按文件夹或文档勾选导出范围。
- 导出时保留原目录结构，将 Wiki 嵌入和 Markdown 相对路径引用的图片、附件复制到 `_resources/`，同时重写引用路径；不同目录中的同名资源不会互相覆盖。
- 外部 URL 保持原样，源 Vault 只读且不会被修改；路径遍历、符号链接逃逸以及将输出目录放在 Vault 内部等危险操作会被拒绝。
- 插件当前位于 `experimental` 通道，适合先在 Vault 副本或小范围目录上验证结果后再进行完整归档。

### 任务与交互体验

- 统一平台任务运行状态、导航锁、退出确认和全局停止流程，避免运行期间误切页面或重复启动任务。
- 增加持久化任务历史、状态筛选、关键词搜索、失败诊断以及继续/重试入口。
- 普通 URL 和目录字段可保存最近 3 条不同输入；凭证、签名 URL 和敏感字段不会进入历史记录。
- 最近输入不再要求先启动任务：字段确认变更、切换平台和关闭应用时都会安全保存，重新打开即可直接选择最近 3 条。
- 旧表单草稿中已保存的普通 URL 会在恢复时迁移到最近输入；本地存储不可用时不再误报保存成功。
- 表单草稿按插件和动作隔离；配置字段、凭证字段及包含敏感信息的 URL 不会被草稿覆盖或持久化。
- 恢复共享目录树的真实层级、父子联动选择、无可导出节点禁用状态和可访问性提示。
- 任务结果区分文档失败、图片/附件资源警告、部分完成、停止和真正完成，避免“进程退出成功”等同于“任务完整成功”。

### 导入与跨平台稳定性

- 飞书、IMA、印象笔记和语雀导入增加源目录边界校验，阻止本地资源路径越界；IMA 同时规范 Windows 短路径/长路径和 macOS `/var` 路径别名，避免合法源目录被误判。
- 语雀导入补齐资源扫描、断点恢复和稳定任务 ID；有道云笔记改进 Cookie 作用域与资源重定向检查。
- Provider 清单增加字段级历史策略、动作级可见性和统一目录适配契约，并强化构建期与运行时校验。

## 发布前验证要求

本版本合入最新 `main` 后必须重新执行完整发布验收；此前仅包含 10 个插件、15 个 Provider 的安装包及其 SHA256 不得复用。

- 运行 `python scripts/quality_check.py`，确保 Python、Node.js、语法、空白符和全部 Plugin/Provider 契约检查通过；共享目录树契约必须覆盖新增的 `obsidian-export`。
- 当前清单应为 11 个插件、16 个 Provider；构建 Windows 解包产物后运行 `python scripts/package_smoke.py --resources wandao_electron/dist/win-unpacked/resources`，确认包内仍能完整发现这 11 个插件和 16 个 Provider。
- 重新验证最近 URL 的 3 条去重记录、字段确认变更、切换平台、关闭应用、旧草稿迁移以及敏感字段不落盘。
- 重新执行真实飞书 Wiki 的 221/221 目录扫描与 Markdown 导出，以及知识星球 Group 的跨任务续跑、新帖子水位线和历史报告验证。
- 使用样例 Vault 验证 Obsidian 文件夹层级、文档选择参数、Markdown/附件复制、引用重写、同名资源隔离和路径边界保护。
- 基于最终提交重新构建 Windows x64 与 macOS Apple Silicon 发行物；由 CI 重新生成 `SHA256SUMS`、SPDX SBOM 和 build provenance，并在干净系统完成安装、插件中心、导出任务与恢复任务抽测。

本地 Windows 候选包验收结果：

- `python scripts/quality_check.py` 通过：Python 252 项、Node.js 97 项，包含 `obsidian-export` 共享目录树契约。
- `python scripts/package_smoke.py --resources wandao_electron/dist/win-unpacked/resources` 通过：11 个插件、16 个 Provider、15 个可执行后端。
- `Wandao Setup 1.3.5.exe` 大小为 113,038,344 字节，本地候选包 SHA256 为 `19F338651564F574B76F365F546265FE6BD594F638846BDFE6C7ABB566CECC4D`；最终 GitHub Release 仍以标签构建生成的 `SHA256SUMS` 为准。

## 安装

- Windows：下载并运行 `Wandao Setup 1.3.5.exe`。
- macOS Apple Silicon：解压发行版 ZIP，将 `Wandao.app` 拖入“应用程序”。如首次打开被 Gatekeeper 拦截，可执行 `xattr -cr /Applications/Wandao.app` 后重试。

## 已知限制

- 当前 Windows 和 macOS 安装包仍未配置商业代码签名证书，系统可能显示发布者或 Gatekeeper 提示。
- 飞书源 Markdown 若引用未随 Wiki 节点提供的相对 `.assets` 文件，正文会正常导出，但缺失资源会作为“部分完成”明确列入报告。
- Obsidian 插件当前不支持 `.canvas`、Dataview/Templater 等动态内容、断点续跑和失败重试；Obsidian 专属语法保持原始 Markdown，不做格式转换。
