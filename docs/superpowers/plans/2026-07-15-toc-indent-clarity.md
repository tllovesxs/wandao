# TOC Indentation Clarity Implementation Plan

> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:subagent-driven-development (recommended) or superpowers:executing-plans to implement this plan task-by-task. Steps use checkbox (`- [ ]`) syntax for tracking.

**Goal:** Make second-level and deeper shared directory-tree nodes visually distinct in Feishu and every other scan-enabled platform without changing scan or selection behavior.

**Architecture:** `renderToc()` computes a pixel indentation value for every tree depth and exposes it through `--toc-indent`. The stylesheet uses that same length for row padding and connector placement. The existing Node static test locks both contracts so all providers that share the renderer inherit the fix.

**Tech Stack:** Electron renderer JavaScript, CSS custom properties, Node.js built-in test runner, Python repository quality script.

## Global Constraints

- Apply the visual behavior through the shared TOC renderer; do not add a Feishu-only branch.
- Use a 48px indentation increment exactly.
- Use only existing CSS color variables; connector decorations must stay non-interactive.
- Preserve TOC scan, parent/child selection, export ID, and disabled-module behavior.
- Test-first: production code must not be changed before the targeted test fails for the intended old contract.
