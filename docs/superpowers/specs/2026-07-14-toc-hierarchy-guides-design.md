# 目录树层级缩进与引导线设计

**日期：** 2026-07-14
**状态：** 已选方案，待实施前复核
**范围：** Electron 渲染层中所有支持“读取目录”的平台

## 背景与根因

平台中心将所有具备 `capabilities.scanToc` 的平台目录交由共享函数 `renderToc()` 渲染，并共用 `.toc-item` 样式。当前渲染器向目录项写入 `--depth`，但 CSS 使用：

```css
calc(10px + var(--depth, 0) * 18px)
```

该乘法形式在当前应用运行的 Chromium/CSS 计算中不生效，`padding-left` 因而回退，父级、子级和更深层级显示在同一列。

这不是语雀平台专属问题。已核对以下具备目录扫描能力的平台都会经过同一渲染路径：知识星球专栏、为知笔记、IMA 导出、飞书导出、阿里云通义听悟、OneNote、印象笔记导出、语雀、有道云笔记；不提供“读取目录”界面的平台不受影响。

## 已确认的体验方案：A

采用“**固定缩进 + 淡色层级引导线**”。

每一级目录增加固定 22px 水平缩进；一级以上节点在左侧显示低对比度的纵向引导线和短横连接线。根节点没有引导线。悬停、选中、半选和父节点批量选择行为保持不变。

## 目标

1. 对所有共享目录树渲染的扫描平台，一次性恢复清晰、稳定的父子层级。
2. 不再依赖 CSS 长度与数字的乘法计算；由 JavaScript 为每个节点生成明确的像素缩进变量。
3. 让长目录中当前节点的父子关系无需依赖标题编号也能识别。
4. 保持按钮的完整可点击区域、现有键盘焦点行为、勾选状态和文档选择参数不变。
5. 添加针对共享渲染契约的静态/单元回归测试，防止以后重新引入无效 CSS 计算。

## 非目标

- 不在本次加入目录折叠/展开状态、搜索、虚拟滚动或拖拽排序。
- 不修改任何平台的目录扫描协议、节点归一化逻辑或导出选择参数。
- 不将本次 1.3.4 第 103 篇的附件 HTTP 403 与目录 UI 修复混为同一改动；附件问题保持独立诊断项。

## 实现设计

### 1. 共享渲染器：`wandao_electron/renderer/app.js`

`renderToc()` 仍递归构造树，但对每个按钮：

- 保留 `--depth` 作为调试/语义信息；
- 增加 `data-depth`（根为 `0`）；
- 增加 `--toc-indent`，值直接为 `depth * 22px`，例如第二层为 `44px`；
- 增加 `toc-depth-0` 等深度类，供 CSS 排除根节点引导线。

这样计算在 JavaScript 完成，CSS 只进行“长度 + 长度”的有效 `calc()`。

### 2. 共享样式：`wandao_electron/renderer/styles.css`

`.toc-item` 改为使用：

```css
padding-left: calc(10px + var(--toc-indent, 0px));
```

并将目录项设为定位上下文：

- 非根项的 `::before` 在缩进的左侧绘制淡纵向线；
- 非根项的 `::after` 绘制连接到复选框附近的短横线；
- 线条不参与点击、不覆盖复选框、不改变按钮高度；
- 高对比度状态下仍保留文字、复选框和计数，因此线条只是辅助而非唯一层级信号。

### 3. 跨平台覆盖说明

平台中心仅在 `provider.capabilities.scanToc` 为真时输出共享 `renderTocShell()`；扫描成功后统一调用 `normalizeTocNodes()` 和 `renderToc(provider.id)`。因此改动共享 renderer 即可同时覆盖所有上述平台，不需要在各插件中复制样式或后端逻辑。

## 测试与验收

1. 在 `tests_js` 添加渲染契约回归测试：检查 `app.js` 为树节点输出 `data-depth` 与 `--toc-indent`，并检查 CSS 使用有效的 `--toc-indent` 长度变量，而不是 `var(--depth) * 18px`。
2. 使用现有多层测试数据验证根、一级子项、二级子项产生 `0px`、`22px`、`44px` 缩进值。
3. 运行：
   - `npm run check`；
   - 新增的 Node 测试；
   - `node --test tests_js/toc_tree.test.js`；
   - 必要时 `python scripts/quality_check.py`。
4. 启动 Electron 源码版，至少在语雀和另一目录扫描平台的界面中读取目录，人工确认缩进、引导线、全选/全不选/反选以及点击父节点批量选择行为。

## 风险与缓解

- **深层目录占用宽度：** 固定 22px 并保留标题省略；不扩大每行最小高度。
- **伪元素干扰点击：** 使用 `pointer-events: none`，只作视觉装饰。
- **共享组件回归影响广：** 先以渲染契约测试锁定输出，再用两个不同平台进行人工验证。

## 实施完成标准

所有目录扫描平台显示明确的逐级缩进和引导线；根节点无引导线；现有选择结果和导出参数不变；源码检查、测试和手动 Electron 验证通过。
