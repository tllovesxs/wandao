# 目录树二级缩进清晰度设计

**日期：** 2026-07-15
**状态：** 已获用户确认，实施中
**范围：** Electron 平台中心中所有使用共享 `renderToc()` 的目录树；飞书为本次验收重点。

## 问题与根因

飞书的目录扫描结果已携带正确的父子关系，且共享渲染器会按递归深度输出 `data-depth` 与 `--toc-indent`。当前每层缩进仅为 30px，连接线使用较浅的 `--border`，使二级目录与根目录的复选框和标题起点距离不足，长标题场景下层级不明显。

这不是飞书读取、目录归一化、限流、导出或选择参数问题；所有启用目录读取的平台都经过同一个渲染器，因此应在共享 UI 层修复。

## 已确认方案

1. 将共享渲染器的每层显式缩进由 30px 提高到 48px。根节点为 0px，一级子节点为 48px，二级子节点为 96px。
2. 保持既有 `toc-depth-N`、`data-depth`、`--depth` 与 `--toc-indent` 契约，避免改变目录数据和选择行为。
3. 将非根节点的竖线和横向连接线改用现有 `--border-strong` 颜色令牌；连接线位置仍从 `--toc-indent` 推导，不覆盖复选框，也不接收指针事件。
4. 更新共享渲染回归测试，先验证测试因旧的 30px / `--border` 实现而失败，再最小化修改生产代码使其通过。
5. 完成自动检查后重新启动源码 Electron，供飞书和其他目录读取平台手工验收。

## 非目标

- 不修改飞书插件、扫描请求、导出请求、限流策略或日志处理。
- 不新增目录折叠、搜索、拖拽、虚拟滚动或平台专属样式分支。
- 不改变复选框选择、父节点批量选择、全选、反选及导出 ID 计算。

## 验收标准

- 二级目录相对根目录至少额外偏移 96px，且三级及更深层级按 48px 递增。
- 非根节点显示更易辨认但不抢占内容的父子连接线。
- 所有扫描目录的平台继续使用同一个 `renderToc()` 路径。
- 目录树回归测试、语法检查与项目质量检查通过。
- 源码 Electron 以 `npm run dev` 启动，并保留本次启动日志。

## 自检

范围只包含共享 renderer、CSS 与既有静态回归测试；没有遗漏任何会改变飞书数据或导出行为的需求，也不存在平台专属重复实现。
