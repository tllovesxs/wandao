# WPS 文档格式分流设计

## 目标

让 WPS 插件采用与项目其他内容平台一致的导出边界：WPS 智慧文档输出 Markdown，普通云文件保留原始格式，设备文档继续排除。

## 范围

只修改 WPS 插件及其测试，不修改其他平台和通用 UI。

## 导出规则

1. 扫描阶段保留 WPS 返回的文件类型信息，用于区分智慧文档与普通文件。
2. 智慧文档直接调用只读内容接口，转换并写入同名 `.md`；不再先尝试下载原始文件。
3. 普通 Word、Excel、图片及其他可下载云文件继续通过 WPS 下载接口保存原始文件。
4. 若扫描数据无法明确判断类型，沿用原文件下载优先、明确允许时才回退 Markdown 的兼容逻辑，避免误把普通文件转成 Markdown。
5. 在线智能表格或尚未上传完成的文件如果不能下载，继续记录准确的单项失败，不中断其他项目。
6. 保持目录层级、断点续传、失败报告和浏览器会话处理不变。

## 成功标准

- 已识别的智慧文档即使存在原始下载地址，也只生成 `.md`。
- 普通文件仍调用下载接口并保留原扩展名。
- 现有 WPS 登录、扫描、失败报告测试不回归。
- 不修改其他平台代码和 UI。
