# 万能导全项目执行计划

## 最终目标

万能导从“多个平台脚本加桌面壳”演进为本地优先、可信、可恢复的知识迁移产品。平台能力全部通过 Plugin v1 提供，核心程序只负责插件发现、任务编排、安全边界、统一界面和本地数据。

## 完成定义

- 飞书、语雀、知识星球、阿里云 Thoughts、印象笔记、有道云笔记、为知笔记、OneNote、ima、Notion 全部位于 `plugins/<plugin-id>/`。
- 每个平台都有合法 `plugin.json`，每个动作都有 Provider v1 清单和契约测试。
- 平台实现不再保存在仓库根目录；根目录同名脚本仅作为兼容入口。
- 新增或删除平台不需要修改 Electron 内置平台注册表、脚本白名单或逐文件打包清单。
- 随应用提供的官方插件开箱可用；签名安装版本可以覆盖内置版本，停用状态可被尊重。
- Python 任务必须输出可校验的 TaskResult v1；缺失或错误结果不得静默成功。
- 日志、报告、历史和 checkpoint 使用统一的 `runId`、`jobId` 与重试关系。
- 插件子进程使用环境变量白名单；凭证不通过命令行或普通日志传播。
- 长任务关闭窗口前必须确认，停止任务能够终止进程树并保留可恢复状态。
- PR 必须通过跨平台质量门禁、Provider 契约、Electron E2E 和安装包冒烟。
- 正式发行物具备校验和、SBOM、provenance；Windows/macOS 签名配置完成后才能标记正式版。

## 执行阶段

### 阶段 0：基线和安全止血

- 固化全平台清单、完成定义和自动化基线。
- 严格任务结果帧，插件环境变量白名单，敏感文件默认忽略与 Secret 扫描。
- 统一 Node 版本并给便携运行时下载增加哈希校验。

### 阶段 1：全平台插件化

- 把平台实现和 Provider 清单迁入插件目录。
- 增加官方内置插件发现，已安装签名插件优先于内置版本。
- 删除渲染层平台注册表和主进程逐平台脚本白名单。
- 打包改为复制核心运行时和插件目录，不再逐个列平台脚本。

### 阶段 2：协议和任务可靠性

- 引入 TaskResult/Event JSON Schema 和严格适配层。
- 建立 TaskManager 状态机、可靠取消、退出确认、重试 lineage 和资源锁。
- 统一任务历史、结构化日志、报告、checkpoint 身份。

### 阶段 3：核心拆分和产品体验

- 把浏览器、认证、HTTP、资源、Markdown、报告和 checkpoint 抽入 `wandao_core`。
- 消除平台脚本互相 import，停止维护旧 Tk GUI。
- 拆分 `main.js` 与 `renderer/app.js`，复杂导入改为有门禁的步骤流。
- 增加表单草稿、字段级错误、任务筛选和大目录树优化。

### 阶段 4：质量、发布和治理

- 增加 lint、类型检查、覆盖率、依赖审计、Secret 扫描和 CodeQL。
- 增加 Electron E2E、打包后 Python smoke、Windows/macOS/Linux 质量矩阵。
- 生成 checksums、SBOM、provenance，保护插件签名工作流和密钥轮换。
- 同步 README、教程、版本说明、SECURITY、CODEOWNERS 和发布/回滚手册。

## 优先级规则

1. 数据丢失、凭证泄漏、静默成功和无法恢复属于 P0。
2. 已有稳定平台的回归优先于新增平台。
3. 新平台先交教程或最小 Provider，达到契约和样例验收后再标记 stable。
4. 核心程序不得新增平台 ID 分支；平台差异必须进入插件声明或插件后端。
5. 所有遥测默认关闭；产品指标优先本地统计或用户明确选择加入。

## 验收指标

- stable Provider 的样例任务成功率不低于 95%，资源完整率不低于 98%。
- 失败任务恢复成功率不低于 90%，退出或崩溃不得丢失已完成文件。
- 所有正式 Provider 都通过 `--help`、动作参数、结果协议和报告 schema 契约测试。
- 高危和严重依赖漏洞为 0；Secret canary 不进入 Git、任务历史、日志或支持包。
- 正式安装包签名验证通过，所有发行资产均有 checksum、SBOM 和 provenance。
