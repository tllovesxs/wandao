# 3天学完一个大型项目，一周拿到 offer？

> 这不是承诺，也不是玄学，而是我想分享的一套“用 AI 学大型项目”的方法。

以前我学一个大型项目，基本都是这个流程：

先看视频。

再看文档。

遇到一个不懂的问题，再把相关文档复制下来问 AI。

听起来好像没什么问题，但真正学起来会发现特别痛苦。

因为大型项目的文档不是孤立的。

你今天问的是“下单流程”，但它可能牵扯到登录鉴权、库存扣减、分布式事务、支付回调、MQ 消息、定时关单、面试题总结。

你只复制当前这一页文档给 AI，它回答得就很局部。

但你要把所有相关文档都复制过去，又太麻烦。

更难受的是，知识是断的。

视频里讲一部分，文档里写一部分，源码里又是另一种实现。

很多时候你不是不努力，而是资料之间没有建立联系。

于是我做了一个项目：**万能导 Wandao**。

它解决的问题很简单：

**把你有权限访问的项目资料，一键导出成本地 Markdown 文档。**

![万能导启动器](images/wandao-launcher.png)

导出之后，你只需要把这些资料放进项目源码目录里，再把我整理好的“项目教学 Skill / 提示词”发给 AI。

这时候你再问 AI：

- 讲一下订单下单流程
- 为什么这里要用 Redis Lua？
- Seata AT 和 TCC 有什么区别？
- 这个项目的面试亮点有哪些？
- 如果面试官问超卖问题，我该怎么回答？

AI 就不再是只看你复制的某一小段内容，而是可以结合整个知识库、项目源码和教学文档一起回答你。

目前工具做了图形化界面，可以选择不同类型的知识库导出，也可以读取目录后选择导出全部内容，或者只导出某些章节。

![万能导多平台导出界面](images/wandao-exporter-guis.png)

它会像一个项目学习导师一样：

- 先看教学文档
- 再看源码实现
- 再串起业务流程
- 再解释技术取舍
- 最后帮你整理面试回答

最关键的是，如果文档里本身就有真实面试题，它也可以帮你一题一题过。

以前是我追着资料跑。

现在是我把资料整理好，让 AI 带着我学。

这也是我做 **万能导** 的原因。

它不是为了“采集资料”，而是为了减少重复复制粘贴，让学习大型项目这件事更完整、更系统。
