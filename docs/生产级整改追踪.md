# 生产级整改追踪

本清单以“可验证关闭”为准：实现、回归测试、质量门禁和用户可观察行为均满足后才可标记完成。状态中的“进行中”表示当前工作分支已有专职修复，不代表可发布。

## P0：数据、凭证与可恢复性

| ID | 项目 | 状态 | 完成标准 |
| --- | --- | --- | --- |
| REL-001 | checkpoint 并发租约 | 进行中 | 活跃任务不可被第二进程恢复；租约过期才可接管；关闭释放租约。 |
| REL-002 | 增量导出完整性 | 待办 | 只有 checkpoint 已完成且 Markdown/资源完整时才跳过；资源失败项必须重试。 |
| REL-003 | 导入幂等性 | 待办 | 语雀、飞书、印象、ima 在远端创建后崩溃时可复用远端 ID，不重复创建。 |
| SEC-001 | 语雀/飞书 Cookie 重定向边界 | 进行中 | 禁止自动跨域复制 Cookie；每跳 HTTPS、域与 Cookie 规则均校验。 |
| SEC-002 | 有道 Cookie 与资源 URL 边界 | 已完成 | 仅受信任 HTTPS 有道域可带 Cookie；跨域跳转重新建无 Cookie 请求。 |
| SEC-003 | 导入本地路径边界 | 部分完成 | 语雀/飞书/印象/ima 已限制；其余文件型导入器持续接入共享解析器。 |
| SEC-004 | 统一受限下载器 | 待办 | 所有远程资源下载拒绝私网/IP、HTTP、危险重定向与超大响应，并原子落盘。 |

## 验收顺序

1. 合并并验证 UX-003/004、REL-001、SEC-001。
2. 以语雀、飞书为样板关闭 REL-002/003、SEC-004，再迁移其余资源型 Provider。
3. 完成表单草稿、任务中心和导入步骤流。
4. 完成架构拆分与 Electron E2E。
5. 最后完成签名、公证、演练和正式发布验收。
