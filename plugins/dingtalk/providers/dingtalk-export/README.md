# 钉钉文档 Markdown 导出

这是一个实验插件，用于导出**本人有权限访问**的钉钉普通文档（`alidoc`）及其目录层级。

## 使用方法

1. 点击“登录并保存会话”，在打开的钉钉网页完成登录；插件只复用自己的浏览器配置目录，不会保存账号密码、Cookie 或 Token 到配置文件。
2. 在钉钉文档页面复制文档、文件夹或知识库根链接。支持 `docs.dingtalk.com/i/nodes/...`、`alidocs.dingtalk.com/i/nodes/...` 以及知识库首页的 `.../i/spaces/.../overview` 形式。
3. 粘贴链接后点击“读取目录”，按层级勾选需要导出的普通文档。若复制的是知识库中的单篇文档链接，可将“文档链接的导出范围”切换为“所在文件夹及子目录”，或“完整知识库根目录（可能较多）”，再读取目录；默认只导出当前文档，避免意外导出同级内容。
4. 选择输出目录并点击“开始导出”。中断后可在任务中心继续，或使用“只重试上次失败项”。

## 当前范围与限制

- 支持：普通钉钉文档（`alidoc`）的标题、段落、标题层级、列表（含附着在标题上的列表）、代码、链接、表格和常见图片。
- 图片会优先保存到同级 `assets/` 目录；无法下载时会保留原图片链接并在导出报告记录失败项。
- 暂不支持：钉钉表格、脑图、白板、PDF/Word 导出、原始附件和导入钉钉。Markdown 无法无损保留字号、颜色、对齐、间距、表格填充色和合并单元格；这些版式不会伪装成已完整转换。
- 复杂卡片、流程图或未知结构会保留一个指向原文的提示链接，不会伪装成已完整转换。
- 钉钉网页接口可能调整。若出现登录失效或目录为空，请重新登录并提交脱敏错误报告。

## 权限与隐私

插件只访问用户主动登录且有权限访问的钉钉内容。请不要用于绕过登录、权限校验或导出他人未授权资料。

## 参考来源与许可证说明

接口行为和文档结构调研参考了 [Microanswer/ding-doc-downloader](https://github.com/Microanswer/ding-doc-downloader)。该仓库当前未提供许可证文件，因此本插件没有复制其源代码、图片或文档；Markdown 渲染、浏览器会话、目录遍历、资源写入和断点续跑均为万能导内独立实现。
