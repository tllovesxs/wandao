# Wandao 桌面端使用说明

## 普通用户

普通用户优先下载 Releases 页面里的发行版：

- Windows：下载 `.exe` 安装包。
- macOS：下载 `.zip` 压缩包，解压后运行应用。

当前 macOS `.zip` 包未进行 Apple 公证。请只从本项目 GitHub Release 下载；若解压并移动到“应用程序”后提示“已损坏，无法打开”，在终端执行：

```bash
xattr -cr /Applications/Wandao.app
```

再重新打开应用。

发行版已内置 Python 运行时和导入导出依赖，普通用户不需要额外安装 Python。

## 源码运行

源码运行和参与开发仍然需要本机 Python 3.10+ 与 Node.js：

```bash
python -m pip install -r requirements.txt
cd wandao_electron
npm install
npm start
```

## 界面说明

左侧是“首页 / 平台中心 / 任务中心 / 教程公告 / 插件中心 / 设置”工作台。进入平台中心后再选择平台与导出、导入或教程动作。

顶部按钮：

- “夜间模式”：切换日间/夜间主题，并记住用户偏好。
- “检查更新”：检测 GitHub Releases 是否有新版本，发现新版后会提示前往下载。

## 导出流程

1. 在左侧展开“导出”，选择平台。
2. 填写知识库入口 URL。
3. 点击“登录并保存凭证”，在浏览器完成登录后回到工具确认。
4. 点击“读取目录”，在目录树里选择要导出的内容。
5. 按需调整“增量导出”、请求延迟、评论区等选项。
6. 点击“开始导出”。

未读取目录时，工具默认导出入口下可识别的全部内容。

印象笔记不需要入口 URL。第一次使用时填写账号和密码，点击“登录并同步”，再读取目录、勾选范围并导出。

语雀导出默认会下载正文图片和附件。私有知识库、公开密码库和企业私有域名的附件通常依赖登录 Cookie，请先完成“登录并保存凭证”。不想下载大附件时，可以在语雀“导出选项”里关闭“同时下载附件”。

ima 知识库导出不需要浏览器登录。填写 ima Client ID 和 API Key 后，先“保存 API 配置”，再“读取目录”，即可按知识库、文件夹或文件勾选导出。普通文件会尽量保存为原文件；ima 笔记类内容受官方 API 限制，会尽量导出为 Markdown 文本。

## 语雀 Markdown 导入

1. 在左侧展开“导入”，选择“语雀 Markdown 导入”。
2. 填写目标语雀知识库 URL。
3. 点击“登录并保存凭证”，在浏览器里登录语雀并确认能打开目标知识库。
4. 回到工具点击“我已完成登录，保存凭证”。
5. 选择本地 Markdown 目录。
6. 先“生成计划”，再“单篇导入测试”，确认正常后“批量导入”。

语雀导入会按本地文件夹创建语雀目录，并上传 Markdown 中的本地图片和附件。再次导入同一批文档时，会根据本地相对路径生成的稳定 slug 更新对应文档。

## 印象笔记 Markdown 导入

1. 在左侧展开“导入”，选择“印象笔记 Markdown 导入”。
2. 第一次使用先填写账号和密码，点击“登录并同步凭证”。如果已经使用过印象笔记导出，可以跳过这一步。
3. 选择本地 Markdown 目录。
4. 填写默认目标笔记本，按需勾选“按本地目录创建笔记本组/笔记本”。
5. 先“扫描目录”，再“单篇导入测试”，确认图片和附件正常后“批量导入”。

印象笔记导入会把 Markdown 中的本地图片和普通附件作为 Resource 写入笔记。复杂表格或平台私有组件可能会被简化。

## ima 知识库导入

1. 在左侧展开“导入”，选择“ima 知识库导入”。
2. 填写 ima Client ID 和 API Key，点击“保存 API 配置”。
3. 点击“读取知识库”，选择目标知识库。
4. 如需导入到已有文件夹，点击“读取文件夹”，在下拉框里选择目标文件夹；不选则导入知识库根目录。
5. 选择本地文件目录。
6. 先“扫描目录”，再“单文件导入测试”，确认正常后“批量导入”。

第一版使用 ima 知识库 OpenAPI 上传文件，支持 Markdown、PDF、Word、PPT、Excel、图片、TXT、Xmind、音频等文件。默认不会把 Markdown 引用到的本地图片和附件重复作为独立文件上传；需要单独导入这些图片/附件时，可以在高级选项中开启。官方 API 暂未提供明确的创建知识库文件夹接口，所以目前可导入到根目录或已有文件夹。

## 飞书 Markdown 导入

1. 在左侧展开“导入”，选择“飞书 Markdown 导入”。
2. 填写目标飞书 Wiki URL 和本地 Markdown 目录。
3. 打开飞书开放平台，创建企业自建应用。
4. 填写 App ID / App Secret，并点击“保存 API 配置”。
5. 点击“初始化开放平台权限”，按页面提示开通权限并发布应用新版本。
6. 点击“检查应用身份”。
7. 点击“授权目标 Wiki 文档应用”，把当前应用加入目标 Wiki 的文档应用列表并设为可编辑。
8. 点击“登录并保存凭证”。
9. 点击“探测目标 Wiki”和“生成计划”。
10. 先做“单篇导入测试”，确认正常后再“批量导入”。

如果遇到 `99991672`，通常是开放平台 API scope 未开通或未发布新版本。

如果遇到 `131006` 或 `no destination parent node permission`，通常是目标 Wiki 父页面没有给当前企业自建应用文档应用写入权限。

## 进度与停止

任务运行时，界面会显示全局进度条和底部运行日志。导出和批量导入会尽量显示已处理数量、百分比和剩余时间。

需要中断时点击“停止”，工具会在安全点结束并保留已完成文件。

## 打包

```bash
npm run build:win
npm run build:mac:x64
npm run build:mac:arm64
```

打包命令会先下载对应平台的 Python standalone 运行时，并安装 `requirements.txt` 里的依赖。Windows 包可在 Windows 本机生成；macOS 包建议在 macOS 或 GitHub Actions 的 macOS runner 生成。
